# 🧰 Банк промптов для системного аналитика

Готовые формулировки для частых задач. Копируй промпт, подставь свой текст вместо `[…]`.
Пополняется по мере находок («приём недели» из дайджеста добавляем сюда).

> ⚠️ ИИ может уверенно ошибаться (см. [[Глоссарий#Галлюцинация|галлюцинации]]). Любой результат — это **черновик**, который проверяет аналитик. Не загружай конфиденциальные данные без разрешения.

---

## 1. Расшифровка встречи → требования
**Когда:** после созвона/интервью есть «простыня» текста, нужно вытащить суть.
```
Ты — опытный системный аналитик. Ниже расшифровка встречи с заказчиком.
1. Выдели список функциональных требований (что система должна делать).
2. Отдельно — нефункциональные (скорость, безопасность, надёжность).
3. Отдельно — открытые вопросы и противоречия, которые надо уточнить.
Пиши кратко, по пунктам, нумеруй. Не придумывай того, чего нет в тексте.

Расшифровка:
[вставь текст]
```
**Совет:** пункт 3 (открытые вопросы) — самое ценное, экономит итерацию с заказчиком.

## 2. Требование → user stories + критерии приёмки
**Когда:** есть описание функции, нужно оформить по стандарту.
```
Преврати описание ниже в user stories в формате
«Как [роль], я хочу [действие], чтобы [ценность]».
Для каждой истории добавь критерии приёмки в формате Given/When/Then
(Дано/Когда/Тогда), включая 1–2 негативных сценария.

Описание функции:
[вставь текст]
```
**Совет:** проси добавлять негативные сценарии — их чаще всего забывают.

## 3. Проверка требований на пробелы и противоречия
**Когда:** черновик требований готов, нужен «свежий взгляд».
```
Выступи в роли дотошного рецензента требований. Найди в тексте ниже:
- пробелы (что не описано, но понадобится),
- противоречия,
- неоднозначные формулировки («быстро», «удобно» — без цифр),
- крайние случаи, которые не учтены.
Дай список с пометкой важности (высокая/средняя/низкая).

Требования:
[вставь текст]
```

## 4. Описание процесса → диаграмма (Mermaid)
**Когда:** нужно быстро нарисовать схему по тексту.
```
По описанию ниже построй диаграмму на языке Mermaid.
Тип: [sequenceDiagram — взаимодействие во времени / flowchart — блок-схема].
Выдай только код Mermaid в блоке, кратко поясни обозначения.

Описание:
[вставь текст]
```
**Совет:** Mermaid рисуется прямо в Obsidian — вставь код в блок ` ```mermaid `. Удобно для черновика перед «чистовой» диаграммой.

## 5. Функция → тест-кейсы
**Когда:** нужно набросать проверки, в т.ч. негативные.
```
Составь тест-кейсы для функции ниже. Колонки: № | Что проверяем |
Шаги | Ожидаемый результат. Покрой: типичный сценарий, граничные значения,
негативные случаи (неверный ввод, нет прав, нет сети).

Функция:
[вставь текст]
```

## 6. Упростить и причесать документацию
**Когда:** текст громоздкий, термины «гуляют».
```
Перепиши текст ниже проще и короче, не теряя смысла.
Сохрани все факты и цифры. Выровняй терминологию (один термин — одно слово).
В конце дай список терминов, которые ты унифицировал.

Текст:
[вставь текст]
```

---

## 7. Вопросы к заказчику по теме
**Когда:** готовишься к встрече, хочешь ничего не забыть.
```
Я аналитик, готовлюсь обсудить с заказчиком [тема/функция].
Составь список уточняющих вопросов, сгруппируй по темам
(пользователи, данные, интеграции, ограничения, приёмка). 10–15 вопросов.
```
